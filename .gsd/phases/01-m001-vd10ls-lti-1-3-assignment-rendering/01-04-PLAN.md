# S04: TA Checkpoint Verification UI

**Goal:** TA opens dashboard, sees student submissions, toggles checkpoint items (Safety Training, Check Box #1, Check Box #2), and the student's effective score updates
**Demo:** TA launches via /dev/launch/ta, sees submissions with auto scores and consistency flags, toggles checkpoints via HTMX, effective total updates live

## Design

- **CheckpointState model already exists** (submission-scoped: submission_id, checkpoint_id, verified, verified_by, verified_at, points) — no migration needed.
- **Effective score = auto-graded score + verified checkpoint points.** `submission.total_score` stays auto-only; effective total computed on read (S05 AGS will post the effective score).
- **Carry-forward on resubmit:** a new submission copies the latest checkpoint states from the student's previous submission, so TA verifications survive resubmission.
- **TA landing:** GET /assignment/{id} with ta/instructor role now shows the submissions dashboard (newest first, one row per submission: student, time, auto score, S03 consistency-flag count, checkpoint toggles, effective total). Old structural preview stays at /ta/assignment/{id}/structure.
- **Toggle endpoint:** POST /ta/submission/{sid}/checkpoint/{cid} flips state (records verified_by/at), returns the updated row fragment for HTMX swap.
- **Student results page** shows checkpoint sections as Verified (+pts) / Pending TA verification, and the score summary splits auto / checkpoints / total.
- Role gating: /ta/* requires session role ta or instructor.

## Must-Haves

- TA sees all submissions for an assignment with per-submission checkpoint toggles
- Toggling updates effective score without losing auto-graded score
- Checkpoint verifications survive student resubmission
- Students see checkpoint status on results page
- Consistency flags from S03 visible to TA per submission
