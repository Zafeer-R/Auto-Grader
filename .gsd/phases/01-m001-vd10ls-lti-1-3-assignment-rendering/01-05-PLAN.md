# S05: AGS Grade Passback to Canvas

**Goal:** After grading + checkpoint verification, the effective score posts to the Canvas gradebook via AGS (held by Canvas's manual posting policy until instructor release)
**Demo:** TA clicks "Post grade" on a dashboard row (or "Post all"); passback status chip shows posted/failed; failures show the error and a retry, never silently dropped

## Constraint

Real Canvas AGS is blocked on the UTD Canvas admin issuing a developer key. S05 therefore implements the full client + persistence + UI against the AGS spec with a mocked transport in tests, plus a `dry_run` mode so the flow is demoable in dev. Flipping `AGS_MODE=live` + real endpoints/key activates it — no code change.

## Design

- **`app/services/ags.py` — AGSClient** per LTI 1.3 AGS spec:
  - OAuth2 client_credentials with signed RS256 JWT assertion (python-jose, tool private key) at the platform token URL, scope `.../lti-ags/scope/score`
  - POST `{lineitem}/scores` with userId (LTI id), scoreGiven, scoreMaximum, activityProgress=Completed, gradingProgress=FullyGraded, ISO timestamp
  - Retry on network errors/5xx with exponential backoff (1s/2s/4s, 3 attempts); 4xx fails immediately; exhaustion raises AGSError — callers persist the failure
  - Modes: `live` / `dry_run` (records payload, no HTTP) / `disabled`
  - Canvas's manual posting policy holds posted scores until instructor release — no special payload needed
- **`GradePassback` model + migration** (one row per submission, upserted): lineitem_url, lti_user_id, score_given/maximum, status (pending/posted/failed/dry_run), attempts, last_error, posted_at
- **Trigger = explicit TA action** ("TA releases grades"): per-row "Post grade" + header "Post all" (latest submission per student). Posting recomputes the effective score at send time. Submit creates a pending row (intent) using the session's AGS lineitem claim.
- **Config:** LTI_TOKEN_URL, LTI_TOOL_PRIVATE_KEY, AGS_MODE (default dry_run). Dev launch injects a fake lineitem so the flow is demoable.
- **Dashboard:** AGS status chip per row (— / pending / posted ✓ / failed ⚠ with error tooltip / dry_run) + post buttons.

## Must-Haves

- Score payload matches AGS spec; token request uses JWT-assertion client_credentials
- Retry with exponential backoff; failures surfaced on dashboard with error detail; retry button; never silently dropped
- Effective score (auto + verified checkpoints) is what posts
- dry_run demoable end-to-end in dev without Canvas

## Verification

- pytest with mocked httpx transport: payload shape, token flow, retry-then-succeed, retry exhaustion, 4xx no-retry, dry_run, disabled
- Template smoke: status chips + buttons render
- Live-mode round-trip against real Canvas deferred (blocked on UTD dev key) — recorded in UAT
